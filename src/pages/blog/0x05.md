最近准备入门 Flutter 搞移动端跨平台开发，所以这两天学习了 Dart 语言，在学习的过程中发现和我已经比较熟悉的 TypeScript 有类似之处，上手特别快，非常容易理解。

其实我在大二时就已经入门学习 TypeScript，但是没有实践场景，也没有造轮子的想法去实践，直到去年大三在字节实习时接触公司的项目后才开始真正使用 TypeScript。在我熟悉 TypeScript 之前，我有过两次失败的学习经历，第一个是大三尝试学习 Java 时，只有 JavaScript 动态语言编程经验的我实在是理解不了 Java 的设计思想；第二个是我之前也尝试学过 Dart，当时对 Dart 的理解是语言设计和 JavaScript 类似，但是不理解为什么要声明类型。

直到我学习并熟悉了 TypeScript，才恍然大悟：类型系统的存在是高效开发的保障之一，TypeScript 的出现拯救了前端工程师。

后来，和开头说的那样，在有 TypeScript 的基础上，我很容易理解了 Dart 的设计和语法，相信以后有时间学习 Java 的时候，我也不会迷茫了，加上我已经有了 Rust 的学习经历和基础知识（Rust 是我学过的学习曲线最陡峭的语言了，目前还在持续学习中），应该能够比较平滑地学习 Java 知识。

回来本文主题 - 前端进阶，作为一名即将毕业入职的准前端工程师，我已经在字节实习快一年了，在实习时团队一直是把我当成正式员工对待来分配工作任务（这似乎也是字节的传统），在这个过程中，我的前端技术突飞猛进，成长了许多，感谢在这个过程中不断学习和挑战的自己。然而我还是觉得我还有许多不足的地方，很多东西我还没有尝试、挑战和突破，这实际和我个人性格有关，我生性虽然积极、兴趣广泛，但是容易三分钟热度，或者说容易陷入遇到失败挫折后的一蹶不振，也不喜欢尝试自己觉得繁琐但是实际有意义的东西，这是我要反思的地方，不然会让我陷入一个长期的固步自封的状态，虽然在不断扩大自己的围墙，但是永远无法打破自己的围墙。

结合自身的前端技术领域，为了实现我的前端技术进阶，我总结了以下进阶方向：

- 开源项目
- 前端基建，Rust 是下一代前端基建的未来
- 跨平台开发
- Chrome DevTool
- Chrome Extension Dev
- VSCode Extension Dev
- 设计模式
- TypeScript Library Design
- 源码阅读
- 前端性能优化
- 前端单元测试
- 一颗沉得住气的心

在今年三月离开字节结束我的实习旅程时，我在飞书上给自己写下了个人签名：不要建设自己的围墙。

希望我不断进阶，不会止步于前端。
